# 度量（Metrics）

## 核心理念

Harness 的收益难度量，因为它的价值经常是"坏事没有发生"。因此度量不追求单一 ROI，而是沿变更生命周期分层观测。

## 分层指标体系

### 本地（Local）— 开发/实验时的反馈质量

| 指标 | 说明 |
|------|------|
| Hook 拦截次数/分布 | 各层 Hook 拦截了多少问题 |
| Agent 自愈率 | 拦截后 Agent 自动修复的比例 |
| 修复循环次数 | 一个问题平均卡几轮才通过 |

### CI — 门禁有效性

| 指标 | 说明 |
|------|------|
| 首次即绿率 | 提交一次就通过 CI 的比例 |
| 可预防失败率 | 被本地 Hook 漏过但被 CI 捕获的比例 |

### 效率 — 人的释放程度

| 指标 | 说明 |
|------|------|
| 并行任务容量 | 一个人同时管理的 Agent 任务数 |

## ACI（AI Collaboration Index）

综合指标，由三个维度加权构成：

| 维度 | 说明 |
|------|------|
| 深度 | Agent 单次会话中完成的任务复杂度 |
| 覆盖 | Agent 能处理的模块/场景比例 |
| 规模 | 同时运行的 Agent 任务数 |

### 四档协作模式

| 等级 | 名称 | 特征 |
|------|------|------|
| 1 | 基础 | Agent 写简单代码段，Human 全程审核 |
| 2 | 有效 | Agent 独立完成模块级任务，Human Review |
| 3 | 深度 | Agent 跨模块工作，Human 定方向 + Review |
| 4 | 全面 | Agent 全流程自治，Human 只做决策 |
