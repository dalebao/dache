# 渐进式披露（Progressive Disclosure）

## 原则

AI 不应一次性读取全部文档。按需分层加载，从最抽象的入口逐步深入到具体实现。

该设计来源于 UX 设计中的渐进式披露（Progressive Disclosure）原则：向用户展示当前任务所必需的最小信息量，高级/细节信息仅在需要时展开。在 AI Context 上下文中，AI 是"用户"，L0-L4 的每层是一次展开。

## 与常见文档体系的对比

| 体系 | 理念 | 与 L0-L4 的对应 |
|------|------|----------------|
| Diátaxis（技术文档四象限） | 按用户目标分：教程/指南/解释/参考 | 教程→ExecPlan，指南→runbooks，解释→topics，参考→代码 |
| Arc42（架构文档模板） | 按架构视图分：场景/构建块/运行时/部署 | L1-L2 可看作 Arc42 的构建块和运行时视图 |
| ADR（架构决策记录） | 按决策记录，Context/Decision/Consequences | 嵌套在 L3 runbooks/adr/ 下 |

## 各层级详情

### L0 — 入口

文件：`AGENTS.md`（Kilo）/ `CLAUDE.md`（Claude Code）/ `.cursorrules`（Cursor）

职责：
- 项目核心约束（必须遵守的规则）
- 路由表：各层级文档位置和用途
- 命令入口

约束：
- 入口薄化原则：不承载具体知识，只做索引和约束
- 最长不超过 100 行

### L1 — 专题

目录：`.context/topics/`

职责：
- 按主题组织的深度知识
- 跨模块的通用概念
- 架构设计原理

### L2 — 模块

目录：`.context/modules/`

职责：
- 单个模块的入口文档
- 本模块特有的命令和约束
- 不重复 L0 的全局规则

### L3 — 稳定事实 / 操作手册

目录：`.context/runbooks/`

职责：
- 经过验证的稳定知识
- 操作步骤（runbook）
- 执行计划沉淀后的结论

### L4 — 可调用 Skill

目录：`.context/skills/`

职责：
- 可被 AI 调用的能力包，封装完整的可复用操作
- 有明确的输入输出接口
- 可直接执行的原子操作

## 加载策略

AI 启动时默认加载 L0。根据任务需要按路由表导航到 L1-L4。
